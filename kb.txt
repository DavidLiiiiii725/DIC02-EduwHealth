## Topic: What Is Machine Learning
Machine learning (ML) is a subfield of artificial intelligence that enables systems to learn from data and improve their performance on tasks without being explicitly programmed for every case.

In traditional programming, a developer writes rules: INPUT + RULES → OUTPUT.
In machine learning, the system discovers rules from examples: INPUT + OUTPUT → RULES (model).

Machine learning is especially valuable when:
- The rules are too complex or numerous to write by hand (e.g., recognising faces)
- The environment changes over time and rules must adapt (e.g., spam filters)
- Patterns exist in large datasets that humans cannot easily identify manually

Core terminology:
- Dataset: a collection of examples used for learning
- Feature (input variable): a measurable property of each example
- Label (target variable): the outcome we want to predict
- Model: a mathematical function learned from data that maps features to predictions
- Training: the process of adjusting model parameters to fit the data

The central challenge in ML is generalisation: a model must perform well on new, unseen data, not just the examples it learned from.

## Topic: The Machine Learning Pipeline
A typical ML project follows a structured pipeline with distinct phases. Understanding this pipeline helps learners see where each concept fits.

Phase 1 — Problem Definition:
- Clarify the business or research question
- Decide whether the problem is classification, regression, clustering, or generation
- Define a success metric (accuracy, RMSE, F1, etc.)

Phase 2 — Data Collection and Exploration:
- Gather raw data from files, databases, APIs, or sensors
- Explore with descriptive statistics and visualisations
- Identify data quality issues (missing values, outliers, class imbalance)

Phase 3 — Data Preprocessing:
- Handle missing values (imputation or deletion)
- Encode categorical variables (one-hot encoding, label encoding)
- Scale/normalise numerical features
- Split into training, validation, and test sets

Phase 4 — Feature Engineering:
- Create new features from existing ones
- Select the most informative features
- Apply dimensionality reduction if needed

Phase 5 — Model Selection and Training:
- Choose candidate algorithms based on problem type and data size
- Train models on the training set
- Tune hyperparameters using the validation set

Phase 6 — Evaluation:
- Measure performance on the held-out test set
- Diagnose errors (confusion matrix, residual plots)
- Check for fairness and bias issues

Phase 7 — Deployment and Monitoring:
- Package the model for production
- Monitor performance drift over time
- Retrain when performance degrades

Understanding the full pipeline prevents a common mistake: jumping directly to modelling without adequate data exploration and preprocessing.

## Topic: Types of Machine Learning
Machine learning is divided into four major paradigms based on the type of feedback available during training.

1. Supervised Learning:
The model is trained on labelled examples — each example has a known correct answer.
Goal: learn a mapping f(X) → Y that generalises to new inputs.
Examples: email spam detection, house price prediction, medical diagnosis.

2. Unsupervised Learning:
The model is trained on unlabelled data and must discover structure on its own.
Goal: identify clusters, patterns, or compressed representations.
Examples: customer segmentation, topic modelling, anomaly detection.

3. Semi-Supervised Learning:
A small amount of labelled data is combined with a large amount of unlabelled data.
Useful when labelling is expensive but raw data is abundant.
Example: labelling medical images is costly; semi-supervised methods leverage unlabelled scans.

4. Reinforcement Learning:
An agent interacts with an environment and learns by receiving rewards or penalties.
Goal: learn a policy that maximises cumulative reward.
Examples: game playing (AlphaGo), robot locomotion, recommendation systems.

Self-Supervised Learning (a growing paradigm):
The model generates its own supervision signal from the structure of the data.
Example: predicting masked words in a sentence (the basis of BERT and GPT).

## Topic: Supervised Learning — Classification
Classification is a supervised learning task where the output is a discrete category.

Binary classification: two classes (e.g., positive / negative, spam / not-spam).
Multi-class classification: more than two mutually exclusive classes (e.g., digit recognition: 0–9).
Multi-label classification: each example can belong to multiple classes simultaneously (e.g., image tagging).

Common algorithms:
- Logistic Regression: models the probability of a class using the sigmoid function; interpretable and fast.
- k-Nearest Neighbours (kNN): classifies a new example by majority vote of its k closest training examples.
- Decision Tree: partitions the feature space using a tree of if-then rules; interpretable but prone to overfitting.
- Random Forest: ensemble of decision trees trained on random data subsets; robust and accurate.
- Support Vector Machine (SVM): finds the hyperplane with maximum margin between classes; effective in high dimensions.
- Neural Networks: powerful for complex patterns (images, text); require more data and compute.

Key evaluation metrics:
- Accuracy: fraction of correct predictions (misleading with imbalanced classes).
- Precision: of all predicted positives, how many are truly positive?
- Recall (Sensitivity): of all true positives, how many did the model detect?
- F1-Score: harmonic mean of precision and recall; useful when classes are imbalanced.
- ROC-AUC: area under the Receiver Operating Characteristic curve; measures discrimination ability.
- Confusion Matrix: table showing true positives, false positives, true negatives, false negatives.

## Topic: Supervised Learning — Regression
Regression is a supervised learning task where the output is a continuous numerical value.

Common regression tasks:
- Predicting house prices from features (size, location, age)
- Forecasting temperature or stock prices from historical data
- Estimating a patient's blood pressure from clinical measurements

Common algorithms:
- Linear Regression: fits a linear relationship y = wᵀx + b; interpretable and fast.
- Ridge Regression (L2): linear regression with a penalty on large weights to reduce overfitting.
- Lasso Regression (L1): linear regression with a sparsity-inducing penalty; performs feature selection.
- Polynomial Regression: extends linear regression with polynomial features for non-linear relationships.
- Decision Tree Regressor: partitions the feature space and predicts the mean of leaf node values.
- Random Forest Regressor: ensemble of regression trees; robust to outliers.
- Gradient Boosting (XGBoost, LightGBM): sequential ensemble that corrects previous errors; state-of-the-art on tabular data.
- Neural Networks: universal function approximators; effective for high-dimensional or structured input.

Key evaluation metrics:
- Mean Absolute Error (MAE): average of absolute differences between predictions and true values.
- Mean Squared Error (MSE): average of squared differences; penalises large errors more heavily.
- Root Mean Squared Error (RMSE): square root of MSE; interpretable in the same units as the target.
- R² Score (Coefficient of Determination): proportion of variance explained by the model (1.0 = perfect fit, 0 = baseline mean).

## Topic: Bias-Variance Tradeoff
The bias-variance tradeoff is one of the most fundamental concepts in machine learning. It explains why models make errors and guides model selection.

Total prediction error = Bias² + Variance + Irreducible Noise

Bias: error from incorrect assumptions in the model.
- High bias → underfitting: the model is too simple to capture the true pattern.
- Example: fitting a straight line to data that is actually curved.

Variance: error from sensitivity to small fluctuations in the training data.
- High variance → overfitting: the model memorises training data noise.
- Example: fitting a very complex polynomial that passes through every training point exactly.

Irreducible noise: inherent randomness in the data that no model can remove.

The tradeoff:
- Simpler models have high bias and low variance.
- Complex models have low bias and high variance.
- The goal is to find the sweet spot that minimises total error on unseen data.

How to diagnose:
- If training error is high → high bias (underfitting). Fix: increase model complexity, add features.
- If training error is low but validation error is high → high variance (overfitting). Fix: reduce complexity, add more data, use regularisation.

## Topic: Overfitting and Regularisation
Overfitting is the most common failure mode in machine learning. A model that overfits has memorised training data patterns (including noise) and fails to generalise.

Signs of overfitting:
- Very low training loss, much higher validation/test loss
- Model makes confident but wrong predictions on new examples
- Adding more features consistently helps training but hurts validation performance

Regularisation techniques:
1. L1 Regularisation (Lasso): adds the sum of absolute weight values to the loss. Encourages sparse weights (some features get weight zero). Useful for feature selection.
2. L2 Regularisation (Ridge): adds the sum of squared weight values to the loss. Shrinks all weights toward zero without zeroing them. Reduces complexity smoothly.
3. Elastic Net: combines L1 and L2 penalties. Useful when many features are correlated.
4. Dropout (Neural Networks): randomly deactivates neurons during training, preventing co-adaptation. A powerful regulariser for deep networks.
5. Early Stopping: stop training when validation loss stops improving. Prevents the model from over-learning noise.
6. Data Augmentation: artificially expand the training set by applying transformations (flipping images, adding noise). Effective in computer vision and NLP.
7. Cross-Validation: estimate generalisation performance more reliably by training and validating on multiple data splits.

## Topic: Cross-Validation and Model Selection
Cross-validation is a technique for estimating how well a model generalises to unseen data by using the training data more efficiently.

Why standard train/test split is insufficient:
A single split may produce a lucky or unlucky evaluation. The result depends heavily on which examples end up in each split.

k-Fold Cross-Validation:
1. Divide the training data into k equal parts (folds).
2. Train on k-1 folds, validate on the remaining fold.
3. Repeat k times, each time using a different fold as the validation set.
4. Average the k validation scores for a stable estimate of generalisation performance.

Common choices: k = 5 or k = 10.

Stratified k-Fold:
Ensures that each fold has the same class proportions as the full dataset. Always use this for classification with imbalanced classes.

Leave-One-Out Cross-Validation (LOOCV):
k = N (number of training examples). Accurate but computationally expensive. Used only when datasets are very small.

Model selection workflow:
1. Use cross-validation to compare multiple algorithms (e.g., logistic regression vs random forest).
2. For the winning algorithm, tune hyperparameters using cross-validation.
3. Train the final model on ALL training data with the best hyperparameters.
4. Evaluate ONCE on the held-out test set. Do not peek at test performance during selection.

## Topic: Data Preprocessing and Cleaning
Raw data is rarely clean. Data preprocessing is often the most time-consuming and impactful step in an ML project.

Handling missing values:
- Deletion: remove examples or features with too many missing values (use when missing data is rare and random).
- Mean/Median/Mode Imputation: replace missing values with the column's central tendency (simple but can distort distributions).
- Model-Based Imputation: predict missing values using other features (more accurate but complex).
- Indicator Flag: add a binary feature marking whether a value was missing (preserves the missingness pattern as a signal).

Handling outliers:
- Detect using z-scores, IQR method, or visualisation (box plots, scatter plots).
- Options: remove, cap/clip to a maximum value, or transform (log, square root).
- Important: do not remove outliers without understanding why they exist.

Encoding categorical variables:
- One-Hot Encoding: creates a binary column for each category. Suitable for nominal categories with no order.
  Risk: high cardinality features (e.g., city names with 1000 cities) create very wide feature matrices.
- Label Encoding: maps categories to integers. Only valid if there is a meaningful ordinal relationship.
- Target Encoding: replace each category with the mean of the target variable. Effective but risks data leakage if not done carefully.

Feature scaling:
- Standardisation (Z-score normalisation): subtract mean, divide by standard deviation. Output has mean 0, std 1. Required for algorithms sensitive to feature magnitude (SVM, kNN, neural networks, regularised linear models).
- Min-Max Normalisation: scales features to [0, 1]. Useful when you need a bounded range.
- Robust Scaling: uses median and interquartile range; less sensitive to outliers than standardisation.

Important: always fit scaler/encoder on training data only, then apply to validation and test data. Fitting on the full dataset causes data leakage.

## Topic: Feature Engineering
Feature engineering is the process of using domain knowledge to create, transform, or select input features that improve model performance.

Why feature engineering matters:
A well-engineered feature can dramatically improve a model's ability to learn a pattern. The best features encode the underlying structure of the problem in a form the model can exploit.

Types of feature engineering:
1. Feature Creation: deriving new variables from existing ones.
   - Date/time: extract day of week, hour, month, quarter, is_weekend.
   - Interaction terms: multiply or combine two features (e.g., room_size × price_per_sqm).
   - Polynomial features: x², x³ to capture non-linear relationships in linear models.
   - Domain-specific: in a medical dataset, body mass index = weight / height².

2. Feature Transformation:
   - Log transform: compresses right-skewed distributions (e.g., income, prices).
   - Square root transform: moderate compression for count data.
   - Binning/Discretisation: convert continuous values to categories (e.g., age groups).

3. Feature Selection:
   - Filter methods: rank features by statistical correlation with the target (chi-squared, mutual information, Pearson correlation). Fast but ignores feature interactions.
   - Wrapper methods: train a model on subsets of features and select the best subset (recursive feature elimination, RFE). More accurate but expensive.
   - Embedded methods: feature selection is built into the model (Lasso's L1 penalty, tree feature importances). Efficient and effective.

Caution: only engineer features using information available at prediction time. Using future information in a feature causes data leakage and inflated performance estimates.

## Topic: Gradient Descent and Optimization
Most ML models are trained by minimising a loss function using gradient descent or a variant.

The loss function measures the difference between model predictions and true labels.
- Mean Squared Error (MSE) for regression: L = (1/n) Σ (ŷᵢ - yᵢ)²
- Cross-Entropy Loss for classification: L = -(1/n) Σ [yᵢ log(ŷᵢ) + (1-yᵢ) log(1-ŷᵢ)]

Gradient descent algorithm:
1. Compute the gradient of the loss with respect to each parameter: ∂L/∂w
2. Update parameters in the direction that reduces the loss: w ← w - η · ∂L/∂w
3. Repeat until convergence.
η (eta) is the learning rate: the step size per update.

Learning rate effects:
- Too large: parameters oscillate or diverge; training becomes unstable.
- Too small: training converges very slowly; gets stuck in flat regions.
- Just right: smooth, steady reduction in loss.

Variants of gradient descent:
- Batch Gradient Descent: compute gradient over the entire dataset per update. Stable but slow on large datasets.
- Stochastic Gradient Descent (SGD): compute gradient using one example per update. Fast and noisy; noise can help escape local minima.
- Mini-Batch Gradient Descent: compute gradient over a small batch (16–256 examples). Best of both worlds; most commonly used in practice.

Advanced optimisers:
- Momentum: accumulates a velocity vector in the direction of persistent gradient, damping oscillations.
- RMSProp: adapts the learning rate per parameter based on the magnitude of recent gradients.
- Adam (Adaptive Moment Estimation): combines momentum and RMSProp. Default choice for deep learning in most cases.
- Learning Rate Schedules: reduce learning rate over time (step decay, cosine annealing, warmup + decay). Critical for stable deep network training.

## Topic: Neural Networks — Fundamentals
Artificial neural networks are models loosely inspired by biological neural circuits. They are the foundation of modern deep learning.

Architecture:
- Input layer: receives raw features (one neuron per feature).
- Hidden layers: one or more intermediate layers that learn progressively abstract representations.
- Output layer: produces predictions (one neuron for binary classification with sigmoid; N neurons for N-class classification with softmax; one neuron for regression).

A single neuron:
output = activation(wᵀx + b)
- wᵀx: weighted sum of inputs
- b: bias term
- activation: non-linear function

Activation functions:
- Sigmoid: σ(x) = 1/(1+e⁻ˣ). Outputs 0–1. Used in binary output layers. Vanishing gradient problem in deep networks.
- Tanh: outputs -1 to 1. Slightly better than sigmoid in hidden layers. Still suffers from vanishing gradients.
- ReLU (Rectified Linear Unit): f(x) = max(0, x). Computationally efficient, avoids vanishing gradients for positive values. Default for hidden layers.
- Leaky ReLU: f(x) = max(0.01x, x). Addresses the "dying ReLU" problem where neurons get stuck at zero.
- Softmax: converts a vector of raw scores to a probability distribution over classes. Used in multi-class output layers.

Why depth matters:
- Shallow networks (1 hidden layer) are universal approximators in theory, but in practice require exponentially more neurons than deep networks for complex functions.
- Deep networks learn hierarchical representations: early layers detect simple features (edges, word patterns), later layers detect complex combinations (objects, semantic meaning).

## Topic: Backpropagation
Backpropagation is the algorithm that makes training deep neural networks feasible. It computes the gradient of the loss with respect to every parameter in the network efficiently.

Conceptual steps:
1. Forward pass: feed the input through the network layer by layer; compute the loss.
2. Backward pass: propagate the gradient of the loss backwards through the network using the chain rule of calculus.
3. Update: adjust each weight by moving in the negative gradient direction (gradient descent).

Why the chain rule?
A neural network is a composition of functions: loss = L(f_N(f_{N-1}(... f_1(x) ...))). The chain rule allows us to decompose the gradient of a complex composition into a product of simpler local gradients.

Computational graphs:
Modern frameworks (PyTorch, TensorFlow) represent networks as directed computational graphs, automatically computing gradients through "autograd" or "autodifferentiation". Practitioners rarely implement backpropagation manually.

Vanishing and exploding gradients:
- Vanishing gradients: gradients become very small as they propagate backward through many layers, effectively preventing early layers from learning.
  Solutions: ReLU activations, batch normalisation, residual connections (skip connections in ResNets).
- Exploding gradients: gradients grow very large, causing unstable training.
  Solutions: gradient clipping (cap gradients above a threshold), careful weight initialisation.

Weight initialisation:
- Random initialisation is essential; all-zero weights cause all neurons to learn the same features.
- Xavier/Glorot initialisation: designed for sigmoid/tanh; scales variance by layer size.
- He initialisation: designed for ReLU; recommended default for most deep networks.

## Topic: Convolutional Neural Networks (CNN)
Convolutional Neural Networks are the dominant architecture for image understanding tasks and have applications in any domain where data has spatial or sequential structure.

Core idea: exploit translation invariance and local structure by sharing weights across spatial positions.

Key layers:
- Convolutional layer: applies a set of learnable filters to the input. Each filter slides across the input and computes a dot product, producing a feature map. Detects local patterns (edges, textures, shapes).
- Pooling layer: reduces spatial resolution by computing a maximum or average over a local region. Introduces limited translation invariance and reduces computation.
- Fully Connected layer: at the end of the network, flattened feature maps are passed through standard dense layers for classification or regression.

Important concepts:
- Filter/Kernel: a small matrix of learnable weights (e.g., 3×3) applied via convolution.
- Padding: adding zeros around the input to control output spatial size. "Same" padding preserves spatial dimensions.
- Stride: the step size when sliding the filter. Stride > 1 reduces spatial resolution.
- Depth: each convolutional layer can have many filters, each detecting a different pattern.

Landmark architectures:
- LeNet-5 (1998): pioneered CNNs for digit recognition.
- AlexNet (2012): deep CNN that won ImageNet; triggered the deep learning revolution.
- VGGNet (2014): deep network using only 3×3 convolutions; widely used as a feature extractor.
- ResNet (2015): introduced residual (skip) connections to train networks of 100+ layers without vanishing gradients.
- EfficientNet (2019): scales depth, width, and resolution jointly for best accuracy/efficiency tradeoff.

Transfer learning from CNNs:
Pre-trained CNN weights (from ImageNet) serve as powerful general visual feature extractors. Fine-tuning a pre-trained CNN on a small dataset is almost always better than training from scratch.

## Topic: Recurrent Neural Networks and Sequence Models
Recurrent Neural Networks (RNNs) are designed for sequential data, where the order of elements matters.

Core idea: process one element at a time while maintaining a hidden state (memory) that carries information from previous elements.

RNN recurrence: hₜ = tanh(Wₓhₓₜ + Wₕhₜ₋₁ + b)

Limitations of basic RNNs:
- Vanishing gradients: gradients across many time steps become negligibly small, preventing learning of long-range dependencies.
- Practical limit: standard RNNs effectively only remember ~10-20 time steps.

Long Short-Term Memory (LSTM):
- Introduced gating mechanisms (forget gate, input gate, output gate) that selectively remember or forget information.
- Cell state acts as a "conveyor belt" that carries information with minimal modification across many steps.
- Learns dependencies spanning hundreds of time steps.

Gated Recurrent Unit (GRU):
- Simplified version of LSTM with fewer gates (reset gate, update gate).
- Comparable performance to LSTM with fewer parameters; often preferred for shorter sequences.

Applications of RNNs/LSTMs:
- Language modelling
- Machine translation
- Speech recognition
- Time series forecasting
- Text generation

Note: Transformers (attention-based models) have largely replaced RNNs for most NLP tasks due to better parallelisation and stronger long-range modelling.

## Topic: Transformer Architecture and Attention
Transformers are the architecture behind modern large language models (BERT, GPT, T5, LLaMA). They rely on self-attention rather than recurrence.

Self-Attention mechanism:
- Each position in a sequence attends to every other position, computing a weighted sum.
- Weights (attention scores) are computed as: Attention(Q, K, V) = softmax(QKᵀ/√dₖ)V
  where Q (Query), K (Key), V (Value) are linear projections of the input.
- Allows capturing arbitrary long-range dependencies with constant computation depth.

Multi-Head Attention:
- Run h attention heads in parallel with different projections.
- Concatenate and project the outputs.
- Each head learns to attend to different aspects of the input simultaneously.

Positional Encoding:
Transformers have no inherent notion of order (unlike RNNs). Positional encoding adds a signal about each token's position to its embedding.

Transformer block:
- Multi-Head Self-Attention
- Add & Norm (residual connection + layer normalisation)
- Feed-Forward Network (two linear layers with ReLU)
- Add & Norm

Encoder vs Decoder:
- Encoder-only (BERT): good for understanding tasks (classification, NER, question answering).
- Decoder-only (GPT): good for generation tasks (text completion, code generation).
- Encoder-Decoder (T5, BART): good for sequence-to-sequence tasks (translation, summarisation).

Scale:
Transformer performance scales powerfully with data and parameter count. GPT-4 has hundreds of billions of parameters; this scale produces emergent capabilities.

## Topic: Transfer Learning and Fine-Tuning
Transfer learning leverages knowledge learned on one task and applies it to a related task, dramatically reducing the need for labelled data.

Why transfer learning works:
- Neural networks trained on large datasets learn general features in early layers (edges, shapes for images; grammar, semantics for text).
- These general features are useful starting points for many downstream tasks.

Common transfer learning strategies:
1. Feature Extraction: freeze the pre-trained model weights; use its output as input features for a new classifier. Fast and effective when the target dataset is small.
2. Fine-Tuning: continue training some or all layers of the pre-trained model on the target task. More powerful but requires more data and care to avoid forgetting.

Practical guidelines:
- Small target dataset + similar to pre-training data → feature extraction.
- Large target dataset → fine-tune more layers.
- Different domain from pre-training data → may need to fine-tune deeper layers.

Pre-trained models commonly used:
- Images: ResNet, EfficientNet, ViT (Vision Transformer) — pre-trained on ImageNet.
- Text: BERT, RoBERTa, GPT-2, T5 — pre-trained on large text corpora.
- Audio: Wav2Vec, Whisper — pre-trained on speech data.

Fine-tuning tips:
- Use a smaller learning rate than when training from scratch (0.0001–0.00001).
- Unfreeze layers gradually from top to bottom.
- Monitor validation loss carefully to detect catastrophic forgetting.

## Topic: Natural Language Processing Fundamentals
Natural Language Processing (NLP) is the field of ML concerned with understanding and generating human language.

Key NLP tasks:
- Text classification: sentiment analysis, topic categorisation, spam detection.
- Named Entity Recognition (NER): identify people, organisations, locations in text.
- Machine Translation: translate text between languages.
- Question Answering: retrieve or generate an answer to a natural language question.
- Text Generation: produce coherent text given a prompt.
- Summarisation: condense a document to its key points.

Text representation:
- Bag-of-Words (BoW): represent a document by word counts; ignores word order; high-dimensional and sparse.
- TF-IDF (Term Frequency-Inverse Document Frequency): weights words by how distinctive they are for a document relative to the corpus; improves on BoW.
- Word Embeddings (Word2Vec, GloVe): map each word to a dense vector that captures semantic similarity (similar words have similar vectors).
- Contextual Embeddings (BERT, GPT): represent each word by a vector that depends on surrounding context; far more powerful than static embeddings.

Tokenisation:
Converting raw text into tokens (sub-word units) that can be processed by a model.
- Byte Pair Encoding (BPE): split words into frequent sub-word pieces. Handles rare words and morphologically rich languages.
- WordPiece: similar to BPE; used by BERT.

The pre-training + fine-tuning paradigm:
1. Pre-train a large language model on massive unlabelled text (masked language modelling, next token prediction).
2. Fine-tune on a small labelled dataset for the target task.
This approach achieves state-of-the-art performance across virtually all NLP tasks.

## Topic: Unsupervised Learning — Clustering
Clustering algorithms group examples such that examples within a group are more similar to each other than to examples in other groups.

k-Means Clustering:
Algorithm:
1. Randomly initialise k cluster centroids.
2. Assign each example to the nearest centroid (by Euclidean distance).
3. Update each centroid to the mean of all examples assigned to it.
4. Repeat until assignments no longer change.

Choosing k:
- Elbow method: plot inertia (within-cluster sum of squared distances) vs k; look for the "elbow" where improvement slows.
- Silhouette score: measures how similar a point is to its own cluster vs the nearest other cluster. Range [-1, 1]; higher is better.

k-Means limitations:
- Assumes spherical clusters of similar size and density.
- Sensitive to outliers and initialisation.
- Must specify k in advance.

DBSCAN (Density-Based Spatial Clustering of Applications with Noise):
- Groups together closely packed points; marks outliers as noise.
- Discovers clusters of arbitrary shape.
- Does not require specifying k in advance.

Hierarchical Clustering:
- Agglomerative: start with each example as its own cluster; repeatedly merge the two nearest clusters.
- Results in a dendrogram that allows inspecting cluster structure at any granularity.

Applications of clustering:
- Customer segmentation for marketing
- Document topic discovery
- Gene expression analysis in bioinformatics
- Anomaly detection (unusual examples form their own small cluster or noise)

## Topic: Dimensionality Reduction
Dimensionality reduction compresses high-dimensional data into a lower-dimensional representation while retaining as much meaningful information as possible.

Why reduce dimensionality?
- High-dimensional data is hard to visualise (curse of dimensionality).
- Removing irrelevant or redundant features can improve model performance and training speed.
- Compression enables storage and transmission efficiency.

Principal Component Analysis (PCA):
- Finds the directions (principal components) along which the data varies most.
- Projects data onto the top-k principal components.
- The first principal component explains the maximum variance; each subsequent one explains the most remaining variance.
- Linear method; excellent for visualisation and preprocessing but cannot capture non-linear structure.

t-SNE (t-distributed Stochastic Neighbour Embedding):
- Non-linear method designed for visualisation (typically 2D or 3D).
- Preserves local neighbourhood structure: similar examples appear close together.
- Computationally expensive; perplexity hyperparameter controls the balance between local and global structure.
- Not suitable for general compression or preprocessing (only visualisation).

UMAP (Uniform Manifold Approximation and Projection):
- Faster than t-SNE; better preserves global structure.
- Increasingly preferred for visualisation and preprocessing.

Autoencoders:
- Neural network trained to reconstruct its input through a bottleneck (encoder → latent code → decoder).
- The latent code is a compressed representation.
- Variational Autoencoders (VAEs) produce a structured latent space useful for generation.

## Topic: Ensemble Methods
Ensemble methods combine multiple models to produce predictions that are more accurate and robust than any single model.

Bagging (Bootstrap Aggregating):
- Train multiple models on random subsets of the training data (sampled with replacement).
- Aggregate predictions by majority vote (classification) or averaging (regression).
- Reduces variance without increasing bias.
- Random Forest is the most popular bagging algorithm.

Random Forest:
- Ensemble of decision trees trained on bagged data subsets.
- Each tree also uses a random subset of features at each split → decorrelates trees.
- Hyperparameters: n_estimators (number of trees), max_depth, max_features.
- Provides feature importance estimates.

Boosting:
- Train models sequentially; each model focuses on correcting errors made by the previous ensemble.
- Reduces bias and variance.
- Gradient Boosting: general framework using gradient descent to optimise the ensemble.

XGBoost (Extreme Gradient Boosting):
- Highly optimised gradient boosting implementation.
- Regularisation (L1 + L2 on tree weights) to prevent overfitting.
- Handles missing values natively.
- State-of-the-art for structured/tabular data competitions.

LightGBM:
- Gradient boosting with histogram-based splitting; faster and more memory-efficient than XGBoost on large datasets.

CatBoost:
- Gradient boosting with native support for categorical features; minimal preprocessing required.

Stacking:
- Train multiple diverse base models.
- Train a "meta-learner" on the base models' predictions.
- Can combine models of very different types (neural network + random forest + SVM).

## Topic: Model Evaluation and Metrics
Choosing the right evaluation metric is critical. The metric should reflect what matters for the specific problem.

Classification metrics:
- Accuracy: (TP + TN) / total. Simple but misleading for imbalanced datasets (99% negative class → trivially 99% accurate).
- Precision: TP / (TP + FP). How many predicted positives are truly positive? Relevant when false positives are costly (e.g., spam filter that blocks legitimate emails).
- Recall (Sensitivity): TP / (TP + FN). How many true positives did we detect? Relevant when false negatives are costly (e.g., cancer screening).
- F1-Score: 2 × (Precision × Recall) / (Precision + Recall). Harmonic mean; useful when both false positives and false negatives matter.
- ROC curve: plots true positive rate vs false positive rate at different classification thresholds.
- AUC-ROC: area under the ROC curve. 1.0 = perfect; 0.5 = random. Threshold-independent measure of discrimination.
- PR-AUC: area under the Precision-Recall curve; more informative than ROC-AUC for highly imbalanced datasets.

Regression metrics:
- MAE: average |ŷ - y|. Robust to outliers; easy to interpret.
- MSE: average (ŷ - y)². Penalises large errors more; differentiable everywhere.
- RMSE: √MSE. In the same units as the target; commonly reported.
- MAPE: mean absolute percentage error. Intuitive (e.g., "5% average error") but undefined when y = 0.
- R²: proportion of variance explained. R² = 0 → model does no better than predicting the mean; R² = 1 → perfect.

Calibration:
A well-calibrated model's predicted probabilities match empirical frequencies. Important in medicine and risk assessment. Assess with calibration curves and Brier score.

## Topic: Hyperparameter Tuning
Hyperparameters are configuration settings of a model or training process that are not learned from data and must be set before training.

Examples of hyperparameters:
- Learning rate, batch size, number of epochs
- Number of layers, number of neurons per layer, dropout rate
- Maximum depth of decision trees, number of trees in random forest
- k in k-Nearest Neighbours, C and kernel in SVM
- L1/L2 regularisation strength

Hyperparameter search strategies:
1. Grid Search: evaluate every combination of a predefined set of hyperparameter values. Exhaustive but exponentially expensive.
2. Random Search: sample combinations randomly. Empirically as good as or better than grid search at a fraction of the cost. Recommended as the baseline.
3. Bayesian Optimisation: model the performance surface and use it to select the next hyperparameter setting intelligently. Most sample-efficient; preferred when each trial is expensive.
4. Hyperband / ASHA: multi-fidelity methods that train promising configurations longer and prune unpromising ones early. Very efficient for deep learning.

Practical tips:
- Tune the most impactful hyperparameter first (often learning rate for deep learning).
- Use logarithmic spacing for learning rate and regularisation strength (e.g., [0.001, 0.01, 0.1, 1]).
- Always tune on the validation set or via cross-validation, never the test set.
- Track all experiments (use MLflow, Weights & Biases, or a simple spreadsheet).

## Topic: Reinforcement Learning Basics
Reinforcement Learning (RL) is a learning paradigm where an agent learns to make decisions by interacting with an environment and receiving rewards or penalties.

Core components:
- Agent: the learner/decision-maker.
- Environment: everything the agent interacts with.
- State (s): a description of the current situation.
- Action (a): what the agent does in a state.
- Reward (r): scalar feedback signal from the environment.
- Policy (π): the agent's strategy; maps states to actions.
- Value Function (V(s)): expected cumulative reward from state s under policy π.
- Q-Function (Q(s,a)): expected cumulative reward for taking action a in state s, then following policy π.

The RL objective: find the policy π that maximises expected cumulative discounted reward.

Key algorithms:
- Q-Learning: model-free algorithm that learns Q-values directly from experience without needing a model of the environment.
- Deep Q-Network (DQN): uses a neural network to approximate Q(s,a). First algorithm to achieve human-level performance on Atari games (DeepMind, 2015).
- Policy Gradient Methods (REINFORCE): directly optimise the policy by gradient ascent on expected reward.
- Actor-Critic Methods (PPO, A3C): combine value-based and policy gradient methods; stable and efficient. PPO is the most widely used deep RL algorithm.

Exploration vs exploitation:
- Exploration: try new actions to learn their value.
- Exploitation: use known high-value actions to maximise immediate reward.
- ε-greedy: with probability ε take a random action (explore), otherwise take the best known action (exploit).

Applications of RL:
- Game playing (AlphaGo, OpenAI Five, AlphaStar)
- Robot control and locomotion
- Recommendation systems
- Algorithm optimisation (e.g., neural architecture search)
- RLHF (Reinforcement Learning from Human Feedback) for aligning language models (ChatGPT)

## Topic: ML Project Workflow and Best Practices
Successful ML projects require disciplined workflow, not just modelling skill.

Project phases and common pitfalls:

1. Problem framing:
   - Clearly define the input and output of the model.
   - Understand how the model's predictions will be used (decision threshold, cost of errors).
   - Pitfall: skipping this step and jumping straight to modelling.

2. Baseline models:
   - Always start with the simplest possible model (linear regression, decision tree, rule-based heuristic).
   - A baseline establishes the minimum performance expected.
   - Pitfall: spending weeks on complex models before establishing a baseline.

3. Error analysis:
   - Look at specific examples where the model makes mistakes.
   - Categorise errors to understand systematic failure modes.
   - Pitfall: relying only on aggregate metrics and missing systematic errors.

4. Iterative improvement:
   - Make one change at a time; measure the effect of each change.
   - Keep a log of what worked and what did not.
   - Pitfall: making multiple changes simultaneously and not knowing which helped.

5. Reproducibility:
   - Set random seeds for all stochastic operations.
   - Version your data, code, and model.
   - Document the exact training command and hyperparameters.
   - Pitfall: getting a good result once and being unable to reproduce it.

6. Data leakage:
   - Data leakage occurs when information from the test set (or future) influences model training.
   - Common sources: preprocessing on the full dataset before splitting, using future features.
   - Consequence: inflated performance estimates that don't hold in production.
   - Prevention: always split data first, then preprocess within each fold.

7. Model deployment considerations:
   - Latency requirements: how quickly must the model respond?
   - Data distribution shift: will the production data distribution differ from training data?
   - Monitoring: implement alerts for performance degradation.

## Topic: Fairness, Bias, and Ethics in Machine Learning
ML models can encode and amplify societal biases if not carefully designed and audited.

Sources of bias:
- Historical bias: the data reflects historical inequities (e.g., hiring decisions biased against women).
- Measurement bias: the proxy variable used to measure a concept is biased (e.g., using criminal records as a proxy for recidivism risk).
- Representation bias: some groups are underrepresented in training data (e.g., medical datasets with predominantly one ethnicity).
- Label bias: human annotators introduce their own biases when labelling data.
- Feedback loops: a biased model influences outcomes that become future training data, compounding the bias.

Fairness definitions (not all can be satisfied simultaneously):
- Demographic Parity: equal positive prediction rates across groups.
- Equalised Odds: equal true positive AND false positive rates across groups.
- Calibration: predicted probabilities equally accurate across groups.

Practical fairness steps:
1. Disaggregate evaluation metrics by demographic subgroup.
2. Identify which errors are most harmful in context.
3. Apply bias mitigation techniques: re-sampling, re-weighting, post-hoc threshold adjustment.
4. Engage with affected communities during development.

Ethical principles for responsible ML:
- Transparency: document model limitations, training data, and known failure modes.
- Accountability: identify who is responsible for model decisions.
- Privacy: protect training data; implement differential privacy where needed.
- Safety: anticipate misuse and include safeguards.
- Human oversight: keep humans in the loop for high-stakes decisions.

## Topic: Common ML Mistakes and Debugging Strategies
Knowing how to diagnose and fix problems is as important as knowing algorithms.

Problem: Model performance is poor on training data.
Cause: Underfitting — model is too simple.
Fix:
- Use a more complex model (add layers, increase depth).
- Add more or better features.
- Train for more epochs.
- Reduce regularisation.

Problem: Model performs well on training but poorly on validation.
Cause: Overfitting — model has memorised training data.
Fix:
- Increase training data.
- Apply regularisation (L1, L2, Dropout, Early Stopping).
- Use a simpler model.
- Apply data augmentation.

Problem: Training loss is not decreasing.
Causes and fixes:
- Learning rate too small: increase it.
- Learning rate too large (loss oscillates or diverges): decrease it.
- Vanishing gradients: use ReLU activations, batch normalisation, He initialisation.
- Exploding gradients: apply gradient clipping.
- Bug in data pipeline: verify that batches are loaded and preprocessed correctly.

Problem: Validation loss decreases then increases (training diverges after a while).
Cause: Overfitting begins after initial generalisation.
Fix: use early stopping; stop training when validation loss stops improving.

Problem: Very slow training.
Fix:
- Use GPU (move tensors to device).
- Increase batch size (use all available GPU memory).
- Use mixed-precision training (float16 instead of float32).
- Profile your data loader; it may be the bottleneck.

Problem: Inconsistent results between runs.
Fix:
- Set random seeds for Python, NumPy, and the framework.
- Use deterministic CUDA operations (at cost of some speed).

Debugging checklist:
□ Is the data loaded and shaped correctly?
□ Are features scaled appropriately?
□ Is there data leakage?
□ Are the loss and metric values in a reasonable range?
□ Is the model architecture correct (output shape, activation function)?
□ Are gradients flowing (check gradient norms)?

## Topic: Learning Strategies for ML Students
Effective learning in machine learning requires deliberate strategies because the field is broad, mathematically demanding, and rapidly evolving.

Principle 1 — Build intuition before formalism:
Understand what an algorithm does conceptually before studying the mathematics.
Example: understand that gradient descent "walks downhill on a loss landscape" before reading gradient formulas.

Principle 2 — Implement to understand:
Reading about backpropagation is insufficient. Implementing it from scratch (even once) reveals the details that notation obscures.

Principle 3 — Start small and scale:
Begin with small datasets and simple models. Get the pipeline working end-to-end. Only add complexity when you have a working baseline.

Principle 4 — Experiment deliberately:
Change one variable at a time. Keep a log of what you changed and what happened.

Principle 5 — Embrace confusion:
Confusion is evidence that you have encountered a concept that does not yet fit your mental model. It is a signal to slow down, not to give up.

Principle 6 — Use multiple resources:
Different textbooks and courses explain the same concept differently. When one explanation does not work for you, seek another.

Principle 7 — Build projects:
End-to-end projects consolidate knowledge and reveal gaps. Choose a topic you are genuinely interested in; motivation sustains the effort required.

Principle 8 — Engage with the community:
Kaggle competitions, paper reading groups, and online forums expose you to approaches and perspectives beyond any single course.

Principle 9 — Teach what you learn:
Explaining a concept to someone else reveals precisely what you do not understand. Use the Feynman technique: try to explain in simple language; where you get stuck, that is your gap.

## Topic: Growth Mindset in Machine Learning
Struggling with machine learning does not indicate lack of ability. It reflects the genuine difficulty of the material.

Machine learning requires:
- Abstract mathematical thinking (linear algebra, calculus, probability theory)
- Programming proficiency (Python, numerical computing, debugging)
- Systems thinking (pipelines, data flow, infrastructure)
- Scientific discipline (hypothesis testing, controlled experiments)
- Domain knowledge (understanding the data and the problem)

Mastery requires integrating all of these simultaneously. This is challenging for everyone, regardless of background.

Growth mindset principles for ML learners:
- Ability is developed through effort and effective strategy, not fixed at birth.
- Difficulty is information: it tells you where to focus effort, not that you should stop.
- Mistakes are essential data points; examining them accelerates learning.
- Other people's apparent ease comes from experience you haven't yet accumulated.

Common negative thought patterns to reframe:
- "I'm not smart enough for this" → "This concept is genuinely hard; what do I need to understand it?"
- "Everyone else seems to understand" → "Everyone else has their own confusions; they are not all visible."
- "I've been stuck for an hour; this is hopeless" → "Being stuck means I'm at the edge of my current understanding. That's exactly where growth happens."
- "I got lucky on that project" → "I applied effective strategies that worked. I can identify what worked and do it again."

Practical actions:
- Set specific, achievable session goals ("understand how softmax is derived" rather than "learn NLP").
- Track small wins; a learning journal builds evidence of progress.
- Seek help before frustration becomes discouragement; asking for help is a skill, not a weakness.

## Topic: Motivation and Self-Efficacy in Technical Learning
Motivation in technical subjects is closely linked to perceived self-efficacy — the belief that you can succeed at a task with effort.

Self-Determination Theory (Deci & Ryan):
Three basic psychological needs support intrinsic motivation:

1. Competence: the need to experience mastery and effectiveness.
   - How to build it in ML: work through concrete exercises that produce visible results; use small datasets where training is fast.
   - Celebrate small victories: "I built my first neural network that converges" is meaningful progress.

2. Autonomy: the need to feel volitional control over your learning.
   - How to build it: choose projects aligned with your interests; drive your own learning path.
   - Personalise your study: if you prefer intuition over formulas, start there.

3. Relatedness: the need to feel connected to others who care about your success.
   - How to build it: join study groups, online communities, pair-program with peers.
   - Teaching others doubles as learning; explaining deepens understanding.

Expectancy-Value Theory:
Motivation is high when:
- Expectancy: you believe you can succeed with effort (self-efficacy is high).
- Value: the task is relevant, interesting, or important to your goals.

Building self-efficacy in ML:
1. Start with tasks slightly above your current ability (challenging but achievable).
2. Break tasks into verifiable sub-steps so each completion is a success signal.
3. Identify and attribute successes to your effort and strategy.
4. Find a mentor or model who succeeded via consistent work, not innate genius.

When motivation drops:
- Return to something you already know and can do. Rebuild momentum.
- Identify whether the drop is due to confusion (need help), boredom (need challenge), or exhaustion (need rest).
- External deadlines and accountability partners can bridge motivation gaps.

## Topic: Managing Cognitive Load While Learning ML
Cognitive load theory explains why some teaching approaches work better than others, and helps learners manage their own mental resources.

Types of cognitive load:
- Intrinsic load: the inherent complexity of the material itself (e.g., backpropagation has many interacting components).
- Extraneous load: unnecessary cognitive work caused by poor presentation (e.g., unclear notation, too many unrelated details at once).
- Germane load: cognitive work that directly builds knowledge (e.g., relating a new concept to a prior one).

Strategies for managing cognitive load in ML:

Reduce extraneous load:
- Use a single consistent notation across all resources.
- Study one concept to clarity before moving to the next.
- Remove distractions from your study environment.
- Use visual diagrams (computation graphs, architecture diagrams) to externalise structure.

Manage intrinsic load:
- Decompose complex topics into subtopics and master each before combining.
- Use worked examples before attempting exercises independently.
- When overwhelmed, return to simpler examples (reduce dimensionality, reduce dataset size).

Increase germane load:
- Actively connect new concepts to known ones: "Attention in Transformers is like a weighted average — how is this related to k-nearest neighbours?"
- Practice retrieval: close notes and write down what you remember about a topic.
- Vary practice contexts: apply the same concept in different datasets or problem settings.

Signs of cognitive overload:
- Staring at code or equations without progress for > 15 minutes
- Reading the same paragraph multiple times without retention
- Irritability and inability to concentrate

Response to overload:
- Take a genuine break (5–20 minutes away from screens).
- Switch to a lower-stakes, known task to rebuild confidence.
- Sleep: memory consolidation occurs during sleep; rest is not wasted time.

## Topic: Common Beginner Misconceptions in Machine Learning
Correcting misconceptions early prevents wasted effort and builds more accurate mental models.

Misconception 1: "More data always helps."
Reality: Data quality matters as much as quantity. 1,000 clean, representative examples often outperform 100,000 noisy, biased ones.

Misconception 2: "More complex models are always better."
Reality: Complexity that exceeds the signal in the data leads to overfitting. Start with simple models. Complexity is a cost to pay only when simpler models are insufficient.

Misconception 3: "High training accuracy means the model is good."
Reality: A model can memorise the training set and fail entirely on new data. Always evaluate on held-out data the model has never seen.

Misconception 4: "If I don't understand the math, I can't do ML."
Reality: Conceptual understanding and practical implementation skill can be developed in parallel with mathematical understanding. Many practitioners build effective systems with intuitive rather than formal mathematical understanding.

Misconception 5: "ML will automatically find the right features."
Reality: Feature engineering is a major source of model improvement. Deep learning reduces manual feature engineering for some data types (images, text), but tabular data still benefits enormously from thoughtful feature design.

Misconception 6: "My model performs well in testing, so it will work in production."
Reality: Distributional shift — differences between training data and real-world data — is one of the most common sources of production failures. Always consider the gap between your training distribution and the deployment environment.

Misconception 7: "Neural networks are always the best choice."
Reality: For structured/tabular data, gradient boosting (XGBoost, LightGBM) consistently matches or beats deep neural networks. Neural networks shine for unstructured data (images, audio, text) and very large datasets.

Misconception 8: "I can't do ML without a GPU."
Reality: Many real problems can be solved on a laptop CPU. Start on small datasets locally; scale to the cloud when necessary.

Misconception 9: "Once trained, a model is done."
Reality: Models degrade as the real world changes (concept drift, data drift). Production ML requires monitoring, retraining, and continuous improvement.

## Topic: Summary of Core ML Principles
Machine learning is the practice of extracting patterns from data. Its power comes from generalisation: learning abstract structure that extends beyond the training examples.

The most important ideas to internalise:

1. Generalisation over memorisation: the goal is always performance on unseen data.
2. Bias-variance tradeoff: complexity must be matched to the available data.
3. The pipeline matters as much as the model: data quality and feature engineering often dominate.
4. Evaluation must be rigorous: use held-out data, the right metrics, and cross-validation.
5. Iterate empirically: make one change at a time; measure everything.
6. Understand your data: most model failures trace back to data problems.
7. Simple models are underrated: always establish a strong baseline before increasing complexity.

For learners:
- Machine learning mastery is a gradual, iterative process.
- Confusion and struggle are part of learning, not evidence of inadequacy.
- Every experienced practitioner made the same mistakes you are making now.
- The field rewards persistence, curiosity, and careful thinking more than raw intelligence.
